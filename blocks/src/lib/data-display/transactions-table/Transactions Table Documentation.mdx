# TransactionsTable

A component for displaying transaction records in a clean and sortable table layout. Ideal for financial dashboards, sales records, or any list of data entries that require tabular representation.

## Overview

The `TransactionsTable` component is designed to present structured transactional data with clarity. It supports customization, responsive design, and optionally styled indicators for status, amounts, and categories.

## Features

- Responsive, scrollable table layout
- Status indicators (e.g., completed, pending, refunded)
- Currency and date formatting
- Customizable columns and data structure
- Optional action buttons per row
- Integrates well into card-based dashboards

## Installation

Install the required packages:

```bash
npm install clsx date-fns @/components/ui/table
```

## Usage Example

```tsx
import { TransactionsTable } from '@/components/TransactionsTable';

const data = [
  {
    id: 'txn-001',
    name: 'SaaS Subscription',
    amount: 4500,
    currency: 'USD',
    status: 'completed',
    date: '2024-03-20',
  },
  {
    id: 'txn-002',
    name: 'Office Supplies',
    amount: 120,
    currency: 'USD',
    status: 'pending',
    date: '2024-03-21',
  },
  {
    id: 'txn-003',
    name: 'Refund - Customer #923',
    amount: -99,
    currency: 'USD',
    status: 'refunded',
    date: '2024-03-22',
  },
];

export default function TransactionsExample() {
  return <TransactionsTable title="Recent Transactions" transactions={data} />;
}
```

## Accessibility

- Table rows are keyboard navigable
- Semantic HTML is used for proper table structure
- Ensure text contrast ratios meet WCAG standards
- Optionally add aria-labels for enhanced screen reader support

## Design Guidelines

- Use positive/neutral/negative color cues for amounts and statuses
- Consider aligning numbers to the right for improved readability
- Group transactions by date or category when dealing with long lists
- Keep column labels short and clear

## Dependencies

This component relies on the following:

- `clsx` for conditional styling
- `date-fns` for date formatting
- `@/components/ui/table` for base table structure and styling
